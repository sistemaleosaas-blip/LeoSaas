# Contribuindo para LeoSaas

Obrigado por se interessar em contribuir para o LeoSaas! 🎉

## Como Contribuir

### 1. Fork o Repositório
Clique no botão "Fork" para criar sua própria cópia do projeto.

### 2. Clone Seu Fork
```bash
git clone https://github.com/seu-usuario/LeoSaas.git
cd LeoSaas
```

### 3. Crie uma Branch
```bash
git checkout -b feature/sua-feature
```

Use nomes descritivos:
- `feature/adiciona-autenticacao`
- `fix/corrige-bug-login`
- `docs/melhora-readme`

### 4. Faça Suas Mudanças
- Escreva código limpo e bem documentado
- Siga o estilo de código do projeto
- Adicione testes para novas funcionalidades

### 5. Commit Suas Mudanças
```bash
git commit -m "tipo: descrição breve da mudança"
```

**Tipos de commit:**
- `feat:` Nova funcionalidade
- `fix:` Correção de bug
- `docs:` Documentação
- `style:` Formatação de código
- `refactor:` Refatoração sem mudança de funcionalidade
- `test:` Testes
- `chore:` Tarefas de configuração

### 6. Push para Seu Fork
```bash
git push origin feature/sua-feature
```

### 7. Abra um Pull Request
Clique em "New Pull Request" e forneça uma descrição clara do que você fez.

## Padrões de Código

### TypeScript
- Use tipos explícitos quando possível
- Evite `any`
- Use interfaces para estruturas de dados

### Formatação
```bash
npm run format   # Formata o código
npm run lint     # Verifica erros
```

## Testes

Sempre adicione testes para novas funcionalidades:
```bash
npm test
```

## Reportando Bugs

Abra uma issue com:
- Descrição clara do problema
- Passos para reproduzir
- Comportamento esperado vs real
- Screenshots se aplicável

## Sugestões de Funcionalidades

Abra uma issue descrevendo:
- O problema que resolve
- Casos de uso
- Exemplos de código

## Dúvidas?

- Abra uma issue com a tag `question`
- Veja a seção de Suporte no README
